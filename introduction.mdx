## Overview

BizNode is your portable, autonomous AI business operator. It runs from a USB drive, VPS, or local machine, handling lead collection, intelligent email responses, Telegram interactions, and network coordination without relying on cloud services. Build your own self-operating digital executive with local AI powered by Ollama and Qwen2.5, complete with a semantic memory engine.

Perfect for small businesses and networks, BizNode evolves its knowledge base, extracts structured data from leads, and integrates with partner nodes via the 1BZ registry.

## Hardware and Software Requirements

BizNode requires minimal resources:

| Requirement | Details |
|-------------|---------|
| CPU | Modern x86_64 (Intel/AMD) |
| RAM | 8GB minimum, 16GB recommended |
| Storage | 20GB+ for memory graph |
| OS | Linux, macOS, Windows (Docker recommended) |
| Network | Internet for initial setup and registry |

It supports Ollama for local AI models like Qwen2.5.

## Quick Start

Get BizNode running in minutes.

<Steps>
  <Step title="Download" icon="download">
    Visit [biznode.1bz.biz/download](https://biznode.1bz.biz/#download) and grab the latest portable package.
  </Step>
  <Step title="Setup Telegram Bot" icon="message-circle">
    Create a bot via BotFather and note your `{BOT_TOKEN}`.
    
    ```bash
    export BOT_TOKEN="your-telegram-bot-token-here"
    ```
  </Step>
  <Step title="Configure Email" icon="mail">
    Set up an email account for the node (e.g., Gmail app password).
    
    <CodeGroup tabs="env,docker">
    ````bash
    # .env file
    EMAIL_USER=your-node@biznode.1bz.biz
    EMAIL_PASS=your-app-password-here
    ````
    
    ````dockerfile
    # docker-compose.yml
    environment:
      - EMAIL_USER=your-node@biznode.1bz.biz
      - EMAIL_PASS=your-app-password-here
    ````
    </CodeGroup>
  </Step>
  <Step title="Run Node" icon="play">
    Launch with Docker or directly:
    
    ````bash
    docker run -d \
      --env-file .env \
      -p 8080:8080 \
      biznode:latest
    ````
    
    Access at `http://localhost:8080`.
  </Step>
</Steps>
